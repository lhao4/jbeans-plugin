# JBeans — 产品需求文档（PRD）

**版本**：v1.0  
**日期**：2026-04  
**产品定位**：IDEA 插件 · Java/Spring/Dubbo 本地方法调试台

---

## 一、产品概述

### 1.1 背景

Java 开发者在本地调试时，面临如下困境：

- 调试 Dubbo 接口需要部署注册中心，流程繁琐
- 想验证某个 Service / DAO 方法，必须先写一个 Controller 或单元测试
- 想测一个底层私有子方法，没有任何工具可以直接调用
- Postman 只能覆盖 HTTP 层，Spring 内部逻辑完全无法触达

### 1.2 产品定位

> **填补 Postman 覆盖不到的那一层**：让开发者无需写任何额外代码，直接在 IDEA 内对 Spring 容器中任意 Bean 方法发起调用，像用 Postman 测 HTTP 接口一样丝滑。

### 1.3 目标用户

- Java 后端开发工程师（Spring Boot / Dubbo 技术栈）
- 需要频繁本地调试 Service 层、DAO 层、工具类的开发者
- 不想因为调试目的而修改项目代码的工程师

### 2.2 F1 — 进程管理

#### 需求描述
插件需要连接到用户本地正在运行的 Spring Boot 进程，并在多进程场景下精准区分目标服务。

#### 功能细节

**进程扫描**
- 插件启动后自动扫描本机所有 Java 进程
- 展示维度：服务名 / PID / 项目路径 / 启动端口
- 刷新频率：用户手动刷新 或 首次打开自动扫描

**进程识别（5重特征）**

| 特征维度 | 数据来源 | 说明 |
|---------|---------|------|
| PID | 系统进程表 | 唯一标识 |
| 项目路径 | JVM 启动参数 | 关联 IDEA 打开的项目 |
| 应用名称 | spring.application.name | Spring 配置 |
| Dubbo 应用名 | dubbo.application.name | Dubbo 配置 |
| 服务端口 | server.port | HTTP 端口 |

**连接状态管理**
- 连接中 / 已连接 / 连接失败 / 进程已退出 四种状态
- 进程意外退出时，自动检测并断开，UI 提示友好文案
- 支持随时切换目标进程

**多进程场景**
- 下拉列表展示所有可选服务：`用户服务(PID:12345) - /projects/user-service`
- 自动模式：识别 IDEA 当前激活项目对应的运行进程
- 手动模式：用户强制选择任意进程

#### 验收标准
- [ ] 2秒内完成本机进程扫描
- [ ] 同时启动 3 个 Spring 服务时，能正确区分并分别连接
- [ ] 进程关闭后 5 秒内 UI 状态自动更新

### 2.3 F2 — 方法搜索

#### 需求描述
提供全局方法搜索能力，开发者输入类名或方法名关键词，快速定位目标方法。

#### 功能细节

**搜索数据来源（双轨并行）**

```
PSI 静态扫描（编译期）
  └── 来源：项目 .java 源码文件
  └── 获取：类名、方法名、参数名、注释、注解
  └── 优点：不需要服务启动，参数名完整

运行时反射枚举（运行期）
  └── 来源：Spring ApplicationContext
  └── 获取：实际存在的 Bean 实例、真实方法签名
  └── 优点：100% 真实，条件注入的 Bean 也能正确处理

两者通过「类名 + 方法签名」关联合并
```

**搜索范围**

| 类型 | 注解标识 | 默认展示 | 备注 |
|------|---------|---------|------|
| Dubbo 接口实现 | @DubboService | ✅ | 核心场景 |
| Spring Service | @Service | ✅ | 高频场景 |
| Spring Component | @Component | ✅ | |
| Repository/DAO | @Repository | ✅ | 直测 SQL 逻辑 |
| Controller | @RestController/@Controller | ✅ | HTTP 层 |
| 普通 Bean | @Bean | ✅ | |
| 静态方法 | 无注解 | 需开启开关 | 工具类场景 |
| private/protected 方法 | — | 需开启开关 | 高级调试 |

**搜索交互**
- 输入框实时模糊搜索，延迟 300ms 触发
- 支持：类名搜索 / 方法名搜索 / 注解类型过滤
- 搜索结果按相关度排序，展示：类名 + 方法签名 + 所属模块

#### 验收标准
- [ ] 3秒内完成 PSI 初始扫描并建立索引
- [ ] 输入关键词 300ms 内展示搜索结果
- [ ] 搜索结果准确，不出现已删除的方法

### 2.4 F3 — 参数自动生成

#### 需求描述
选中方法后，根据参数类型自动递归生成可编辑的 JSON 参数模板，用户只需填值，无需手写结构。

#### 功能细节

**类型处理规则**

```
基础类型
  String          →  ""
  int/Integer     →  0
  long/Long       →  0
  boolean/Boolean →  false
  BigDecimal      →  0
  Date            →  "2024-01-01 00:00:00"

集合类型
  List<T>         →  [ {递归展开 T 的结构} ]
  Set<T>          →  [ {递归展开 T 的结构} ]
  Map<K,V>        →  { "key": {递归展开 V 的结构} }
  T[]             →  [ {递归展开 T 的结构} ]

自定义 POJO
  →  递归展开所有字段（含父类字段）
  →  识别 @JsonProperty 等注解重命名

泛型
  List<OrderItem> →  PSI 优先读取源码中的泛型信息
  Result<T>       →  T 不确定时填 {}，加注释提示
```

**边界情况处理**

| 场景 | 处理方案 |
|------|---------|
| 循环引用（A 包含 A） | 记录已解析类型，重复出现填 null |
| 最大嵌套深度 | 默认 5 层，超出截断并提示 |
| 父类字段 | 向上遍历继承链，合并所有字段 |
| @JsonIgnore 字段 | 跳过，不生成 |
| @NotNull 字段 | JSON 注释标注必填 |
| 接口/抽象类参数 | 填 {}，提示用户手动补全具体类型 |

**编辑体验**
- JSON 编辑区支持语法高亮
- 实时语法校验，格式错误红色提示
- 一键格式化（Ctrl+Shift+F）
- 支持多参数方法（生成参数列表结构）

#### 验收标准
- [ ] 5层嵌套 POJO 能正确递归生成
- [ ] List<自定义类型> 能正确识别泛型并展开
- [ ] 循环引用不触发栈溢出

### 2.5 F4 — Bean 方法调用

#### 需求描述
通过 JVM Attach + 反射，对目标进程中的 Spring Bean 实例方法发起调用，返回结果格式化展示。

#### 功能细节

**调用流程**
```
用户点击 Invoke
  → 参数 JSON 校验
  → JSON 反序列化为方法入参对象（支持嵌套/泛型）
  → JVM Attach 挂载目标进程
  → 从 ApplicationContext 获取 Bean 实例
  → 反射执行方法
  → 序列化返回值为 JSON
  → 高亮展示结果
```

**结果展示**
- 成功：绿色状态标识 + 耗时 + JSON 高亮折叠展示
- 失败：红色状态标识 + 异常类型 + 异常信息 + 堆栈（可展开）
- 调用中：按钮 loading 动画 + 禁用防重复点击

**异常分类处理**

| 异常类型 | 展示方式 |
|---------|---------|
| 参数 JSON 格式错误 | 编辑区红色标注 + 提示 |
| 参数类型不匹配 | 提示字段名和期望类型 |
| Bean 不存在 | 提示该类未被 Spring 管理 |
| 业务异常 | 展示异常类型 + message |
| 进程已断开 | 提示重新连接 |

#### 验收标准
- [ ] 本地调用响应时间 ≤ 500ms（不含业务逻辑本身耗时）
- [ ] 业务异常能完整展示，不丢失堆栈信息
- [ ] 调用过程中点击 Cancel 能中断

### 2.6 F5 — HTTP 接口测试

#### 需求描述
扫描项目中的 @Controller / @RestController，自动提取路由信息，内置 HTTP Client 一键发送请求。

#### 功能细节

**扫描范围**
- @GetMapping / @PostMapping / @PutMapping / @DeleteMapping / @RequestMapping
- 提取：HTTP 方法、路径（含类级别前缀）、参数注解

**参数自动填充**

| 注解 | 处理方式 |
|------|---------|
| @RequestBody | 递归生成 JSON Body 模板 |
| @RequestParam | 生成 Query 参数列表 |
| @PathVariable | 生成路径占位符输入框 |
| @RequestHeader | 生成 Header 输入项 |

**请求配置**
- Base URL 可配置（默认 localhost + 自动识别端口）
- 支持添加公共 Header（如 Authorization Token）
- 支持环境变量（{{token}} 语法）

#### 验收标准
- [ ] Spring MVC 注解能完整提取路由信息
- [ ] 路径参数 /api/order/{id} 能正确识别并生成输入框

---

### 2.7 F6 — 静态方法支持

#### 需求描述
支持调用未被 Spring 管理的静态方法，如工具类、算法类、枚举方法。

#### 功能细节

**查找方式**
- 搜索框输入类名，从 PSI 索引 + JVM 已加载类列表中匹配
- 展示类的所有 public static 方法（可开启显示 private static）

**调用方式**
- 无需 Bean 实例，反射 method.invoke(null, args) 直接调用
- 参数生成逻辑与 Bean 方法完全一致

#### 验收标准
- [ ] 正确识别并调用 public static 方法
- [ ] 非 Spring 管理的普通工具类能被搜索到

---

### 2.8 F7 — 历史记录

#### 需求描述
保存每次调用的完整记录（目标方法 + 入参 + 出参 + 耗时），支持快速重放。

#### 功能细节
- 按调用时间倒序展示
- 每条记录展示：方法全名 + 调用时间 + 状态（成功/失败）+ 耗时
- 点击历史记录：自动填充对应方法和参数，一键重放
- 支持收藏常用调用（标星）
- 最多保留最近 200 条记录（本地持久化）

## 三、非功能需求

### 3.1 兼容性

| 维度 | 要求 |
|------|------|
| IDEA 版本 | 2022.3 及以上 |
| JDK 版本 | 8 / 11 / 17（主流版本全覆盖） |
| Spring Boot | 2.x / 3.x |
| Dubbo 版本 | Apache Dubbo 2.7.x / 3.x，Alibaba Dubbo 2.6.x |
| 操作系统 | macOS / Windows / Linux |

### 3.2 性能要求

| 操作 | 目标耗时 |
|------|---------|
| 插件启动 + PSI 扫描 | ≤ 3s |
| 进程扫描 | ≤ 2s |
| 连接目标进程 | ≤ 3s |
| 加载接口列表 | ≤ 2s |
| 本地方法调用 | ≤ 500ms（不含业务耗时）|
| 搜索响应 | ≤ 300ms |

### 3.3 安全性
- 全程不修改目标项目任何代码、依赖、配置
- Agent 注入仅存在于内存，进程关闭后无任何残留
- 不记录任何敏感数据到磁盘（可配置关闭历史记录）

### 3.4 稳定性
- 目标进程崩溃后，插件不受影响，状态自动重置
- 参数反序列化失败时，给出明确字段级错误提示
- 调用超时（默认 30s）自动中断，不阻塞 UI 线程
